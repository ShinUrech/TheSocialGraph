Second-Degree Connection Value in the Last.fm UK Graph

Shin Urech (327245) / Joyti Goel (325374) / Ahmed Chaouachi (346447)

The corrected project direction is to visualize the value of second-degree connections: not only who a person already knows, but who becomes reachable through one mutual friend and why that new connection might matter.

75,969 connected users
389,639 unique undirected friendships
61,826 users with a usable age group
2 hops core design unit

Dataset

Primary dataset: Last.fm UK User Graph Dataset, published on Zenodo by the University of Pisa / CNR. The local project workspace contains the files network, UsersData_anonymized, ArtistTags, ArtistsMap, and Tags.

File Role in the project
network Friendship edge list. After cleaning duplicate and reciprocal rows, it yields 389,639 unique undirected friendships.
UsersData_anonymized User-level demographics: age, gender, country, plus registration date and playcount metadata.
ArtistTags and ArtistsMap Artist and tag metadata. Useful for future music-taste enrichment, but not sufficient on their own to infer user-level taste.
Tags Global tag counts. Useful for understanding the tag vocabulary, but not enough to match taste to individual users.

Data Quality Assessment

The dataset is strong enough for a network-centered project: it is anonymized, well structured, and the friendship graph is explicit rather than inferred. The user table gives us directly usable attributes for age and country-based matching.

Important correction: the current local files do not include a user-to-artist listening table. That means we cannot compute each user's dominant genre from the available workspace alone. Music-taste matching remains part of the vision, but it is a future extension that requires the missing listening-history file.

Preprocessing Requirements

  • Deduplicate the friendship graph, because the raw file contains reciprocal repeats.
  • Clean missing ages and convert valid ages into broader age groups for matching.
  • Standardize country codes and mark incomplete user profiles clearly.
  • Compute second-degree candidate sets for each user by expanding one hop beyond direct friends.
  • Rank candidate second-degree matches by shared country, shared age group, and mutual-friend count.

What We Are Trying To Show

The visualization centers on a simple but powerful social idea: our immediate network is only part of the story. Through one mutual friend, we can reach a much wider set of people, and some of those people are more meaningful than others because they share traits with us.

In the corrected implementation, those shared traits are currently country and age group, because those are the traits supported directly by the available files. The final vision still includes shared music taste, but that part is explicitly deferred until the missing listening data is added.

Project Overview

  • Select one person as the ego node.
  • Highlight their direct friends as first-degree connections.
  • Reveal friends-of-friends as second-degree candidates.
  • Filter the second-degree layer by shared country and/or shared age group.
  • Rank or cap the visible candidates so the interaction stays readable.

Motivation

The motivation remains exactly the same as your original statement: show how existing networks can create new opportunities for connection. The corrected version simply grounds that idea in what the local data can currently support with honesty.

Target Audience

The target audience is broad and non-technical. The project should feel more like an intuitive social discovery experience than a researcher dashboard.

Exploratory Data Analysis Direction

  • Measure degree and second-degree distributions to show how quickly reach expands after one hop.
  • Measure the share of second-degree candidates who also match the ego user on age group and country.
  • Profile a few readable exemplar users to use in the prototype.
  • Use these findings to decide how many candidates to show, how to rank them, and which filters matter most.

Related Work And Originality

Prior Last.fm work is usually recommendation-oriented: users, artists, and tags become features for a model. Our approach is different because the graph itself becomes the story. We are not asking what a user should listen to next; we are asking who becomes meaningfully reachable through one mutual friend.

The originality lies in combining the network structure with interpretable shared attributes and presenting the result as a human narrative instead of a machine-learning benchmark.

Inspiration

The project is inspired by social network visualizations that keep clusters readable and emphasize meaningful structure, especially Martin Grandjean's work on social graph storytelling. The goal is to adapt that readability to a smaller, more personal question: who is one meaningful hop away from me?
